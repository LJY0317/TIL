# S14P-38 지도 기반 위치 추정

## 작업 일시

- 2026-03-17

## 작업 목표

- 저장된 `greenhouse_map.yaml`을 기준으로 `map_server`와 `amcl`을 함께 실행
- Gazebo 시뮬레이션 로봇이 임시 정적 `map -> odom` 대신 AMCL이 계산한 위치 추정을 사용하도록 전환
- RViz2에서 초기 위치를 지정하면 정적 지도 위에서 로봇 자세가 안정적으로 수렴하는 실행 경로를 제공

## 반영 내용

### 1. AMCL 파라미터 파일 추가

- 파일: `agribot_ws/src/agribot_navigation/config/amcl.yaml`
- 내용:
  - `base_frame_id = base_link`
  - `odom_frame_id = odom`
  - `global_frame_id = map`
  - `scan_topic = /agribot/lidar/scan`
  - Gazebo LiDAR 범위와 맞춘 `laser_min_range = 0.12`, `laser_max_range = 10.0`
- 의도:
  - 현재 로봇 TF 트리와 센서 토픽에 바로 맞는 최소 AMCL 설정을 제공
  - 세부 주행 튜닝은 후속 `S14P-204`에서 분리

### 2. localization 전용 launch 추가

- 파일: `agribot_ws/src/agribot_navigation/launch/localization.launch.py`
- 실행 구성:
  - `agribot_description/spawn_agribot.launch.py`
  - `nav2_map_server/map_server`
  - `nav2_amcl/amcl`
  - `nav2_lifecycle_manager/lifecycle_manager`
  - `rviz2`
- 핵심 처리:
  - `publish_map_to_odom_tf:=false`로 설정해 임시 정적 TF를 끄고 AMCL이 `map -> odom`을 발행하도록 전환
  - 저장 지도 기본값을 `maps/greenhouse_map.yaml`로 연결

### 3. 패키지 의존성과 사용 가이드 보강

- 파일:
  - `agribot_ws/src/agribot_navigation/package.xml`
  - `agribot_ws/src/agribot_navigation/maps/README.md`
- 추가 내용:
  - `nav2_amcl`, `nav2_map_server` 실행 의존성 명시
  - localization 실행 명령과 RViz 초기 위치 지정 절차 문서화

## 실행 및 확인 방법

```bash
cd ~/SSAFY/S14P21A602/agribot_ws
source /opt/ros/jazzy/setup.bash
source install/setup.bash
export HOME=/tmp/codex-home
export ROS_LOG_DIR=/tmp/ros_logs

ros2 launch agribot_navigation localization.launch.py
```

RViz2에서:

- Fixed Frame이 `map`인지 확인
- `2D Pose Estimate`로 로봇의 초기 위치와 방향 지정
- `/map`, `/tf`, `/agribot/lidar/scan`이 함께 보이는지 확인

추가 확인 명령:

```bash
ros2 topic echo --once /particle_cloud
ros2 topic echo --once /amcl_pose
ros2 run tf2_ros tf2_echo map base_link
```

## 현재 제한 사항

- 현재 값은 localization 시작선이며, 실제 통로 주행 안정화용 costmap/planner/controller 조정은 `S14P-204` 범위
- AMCL 수렴 품질은 `/agribot/lidar/scan` 품질과 초기 위치 추정 정확도에 직접 영향을 받음
- 지도 파일 자체를 다시 만들거나 해상도를 바꾸면 초기 위치와 파라미터를 함께 재검토하는 편이 안전함
