## 반영 내용

### 1. Nav2 파라미터 파일 추가

- 파일: `agribot_ws/src/agribot_navigation/config/nav2_params.yaml`
- 핵심 조합:
  - Global Planner: `nav2_smac_planner::SmacPlanner2D`
  - Local Controller: `nav2_regulated_pure_pursuit_controller::RegulatedPurePursuitController`
  - Recovery: `spin`, `backup`, `wait`
- 로봇 형상 반영:
  - footprint를 `[[0.24, 0.20], [0.24, -0.20], [-0.24, -0.20], [-0.24, 0.20]]`로 설정
  - `model.sdf`의 `0.4 x 0.3 m` chassis와 바퀴 폭, 센서 여유를 감안해 planner/controller가 벽 간격을 보수적으로 판단하도록 조정
- costmap 기준:
  - local costmap: `4.0 m x 4.0 m`, `0.05 m` resolution
  - global costmap: `0.10 m` resolution
  - `inflation_radius = 0.35`, `cost_scaling_factor = 4.0`
- 주행 제어 기준:
  - `desired_linear_vel = 0.28 m/s`
  - `xy_goal_tolerance = 0.15`, `yaw_goal_tolerance = 0.15`
  - `rotate_to_heading_angular_vel = 0.80`
- 의도:
  - global costmap은 저장된 맵 해상도(`0.10 m`)에 맞춰 과도한 재샘플링과 계산량을 줄이고
  - local costmap만 더 촘촘하게 유지해 레이저 기반 회피와 통로 추종을 안정화

## 검증 결과

### 1. 패키지 빌드

- 명령: `colcon build --symlink-install --packages-select agribot_navigation`
- 결과:
  - `agribot_navigation` 빌드 성공

### 2. launch 인자 파싱

- 명령: `ros2 launch agribot_navigation navigation.launch.py --show-args`
- 결과:
  - `navigation.launch.py`가 정상 로드되고
  - `map`, `localization_params_file`, `nav2_params_file`, `use_rviz`, `autostart` 등 주요 인자가 모두 노출됨

### 3. 실제 기동 확인

- 명령: `timeout 20s ros2 launch agribot_navigation navigation.launch.py use_rviz:=false`
- 결과:
  - `controller_server`, `planner_server`, `smoother_server`, `behavior_server`, `bt_navigator`, `waypoint_follower`가 생성 및 configure 단계까지 정상 진행
  - `SmacPlanner2D`, `RegulatedPurePursuitController`, recovery plugin(`spin`, `backup`, `wait`) 로딩 확인
  - 초기에 `local_costmap.width/height` 타입 문제를 한 번 수정했고, 수정 후에는 파라미터 로딩 에러 없이 진행됨
  - 다만 `farm_world.sdf`에서 Gazebo 물리 부하가 커 `odom -> base_link` TF가 늦어져 `controller_server` activation이 대기 상태로 남았음

## 현재 제한 사항

- 현재 값은 온실 통로 주행의 첫 안정화 세트이며, 실제 waypoint 순찰 좌표와 접근/복귀 로직은 후속 `S14P-205`, `S14P-206` 범위
- global costmap은 저장 맵 해상도에 맞춰 `0.10 m`로 둔 상태라, 맵을 `0.05 m` 해상도로 다시 만들면 planner 파라미터도 함께 재검토하는 편이 안전함
- recovery는 `spin`, `backup`, `wait`만 두었으므로, 복잡한 협소 구역 재시도 전략은 향후 waypoint/mission 단계에서 추가 판단이 필요함
- `farm_world.sdf`는 여전히 triangle-mesh collision 부하가 커서, 실제 목표점 이동 성공률 검증은 `odom`과 TF가 더 안정적인 환경에서 한 번 더 확인하는 편이 안전함
