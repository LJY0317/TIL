# S14P-35 식물·토마토 ID 정의

## 작업 일시

- 2026-03-17

## 작업 목표

- 시뮬레이션 월드의 개별 식물과 토마토 과실에 대해 외부 시스템에서 재사용 가능한 canonical ID를 정의
- Gazebo 모델 이름과 별개로 perception, control, IoT, backend가 공통으로 쓰는 ID 계약을 확정
- `CropStatus` 메시지에 식별 필드를 반영해 이후 상태 발행 시 같은 기준을 쓰게 정리

## ID 정의 원칙

### 1. 구역 기준

- 현재 운영 구역은 `greenhouse_01` 하나만 사용
- canonical ID의 zone prefix는 `gh01`

### 2. 식물 ID 규칙

- 형식: `gh01_plant_<nn>`
- 예시: `gh01_plant_01`
- 식물 상태를 발행할 때 `crop_id == plant_id`

### 3. 토마토 과실 ID 규칙

- 형식: `<plant_id>_tomato_<nn>`
- 예시: `gh01_plant_01_tomato_01`
- 토마토 상태를 발행할 때 `crop_id == tomato_id`
- `plant_id`에는 부모 식물 ID를 함께 넣고, `tomato_id`는 과실 상태일 때만 채움

### 4. Gazebo 모델 이름 사용 기준

- `tomato_plant_1`, `tomato_fruit_9` 같은 Gazebo include name은 시뮬레이션 내부 식별자일 뿐 외부 계약 ID로 사용하지 않음
- 외부 계약은 canonical ID를 사용하고, Gazebo 모델 이름은 `source_model_name`으로만 보존
- 현재 월드 모델과 canonical ID의 연결은 `agribot_description/config/crop_instances.yaml`에서 관리

## 반영 내용

### 1. 단일 소스 레지스트리 추가

- 파일: `agribot_ws/src/agribot_description/config/crop_instances.yaml`
- 내용:
  - zone / ID 패턴 정의
  - 식물 10개 canonical ID
  - 토마토 과실 10개 canonical ID
  - 각 개체의 Gazebo 모델명과 pose
  - 토마토 과실의 부모 식물 ID

### 2. `CropStatus` 메시지 확장

- 파일: `agribot_ws/src/agribot_interfaces/msg/CropStatus.msg`
- 추가 필드:
  - `header`
  - `crop_id`
  - `zone_id`
  - `plant_id`
  - `tomato_id`
  - `source_model_name`
  - `crop_type`

## `CropStatus` 사용 규칙

### 식물 상태 예시

```text
crop_id = "gh01_plant_01"
zone_id = "greenhouse_01"
plant_id = "gh01_plant_01"
tomato_id = ""
source_model_name = "tomato_plant_1"
crop_name = "tomato"
crop_type = "plant"
```

### 토마토 상태 예시

```text
crop_id = "gh01_plant_01_tomato_01"
zone_id = "greenhouse_01"
plant_id = "gh01_plant_01"
tomato_id = "gh01_plant_01_tomato_01"
source_model_name = "tomato_fruit_9"
crop_name = "tomato"
crop_type = "tomato_fruit"
```
