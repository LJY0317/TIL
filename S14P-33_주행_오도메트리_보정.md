# S14P-33 주행·오도메트리 보정

## 작업 일시

- 2026-03-17

## 작업 목표

- `agribot`의 Differential Drive 파라미터를 실제 바퀴 치수와 맞춘 상태에서 더 안정적으로 응답하도록 조정
- `/cmd_vel`이 끊겼을 때 로봇이 계속 주행하지 않도록 watchdog을 추가
- 무거운 온실 메쉬 충돌과 분리된 calibration world에서 `/odom` 증가와 직진 편차를 검증할 수 있게 구성

## 반영 내용

### 1. Differential Drive 파라미터 보정

- 파일: `agribot_ws/src/agribot_description/models/agribot/model.sdf`
- 유지한 값:
  - `wheel_separation = 0.35`
  - `wheel_radius = 0.06`
- 이유:
  - 실제 wheel pose가 `y=±0.175`라서 중심 간 거리가 `0.35 m`
  - wheel collision/visual 반지름이 모두 `0.06 m`
- 조정한 값:
  - `max_linear_velocity = 0.40`
  - `max_linear_acceleration = ±0.60`
  - `max_linear_jerk = ±1.50`
  - `max_angular_velocity = ±1.20`
  - `max_angular_acceleration = ±1.20`
  - `max_angular_jerk = ±2.50`
  - `odom_publish_frequency = 50`
- 효과:
  - teleop 직진 시 급가속/급회전 overshoot를 줄이고 `/odom` 갱신을 50 Hz로 고정

### 2. `cmd_vel` watchdog 추가

- 파일:
  - `agribot_ws/src/agribot_description/agribot_description/cmd_vel_watchdog.py`
  - `agribot_ws/src/agribot_description/launch/spawn_agribot.launch.py`
  - `agribot_ws/src/agribot_description/setup.py`
- 동작:
  - 입력 `/cmd_vel`
  - 출력 `/cmd_vel_safe`
  - 발행 주기 `20 Hz`
  - timeout `0.25 s`
- 이유:
  - Gazebo DiffDrive는 마지막 속도 명령을 유지하므로, teleop 또는 스크립트가 종료된 뒤에도 로봇이 계속 움직일 수 있음
  - watchdog이 stale command를 zero twist로 바꿔 안전하게 정지시킴

### 3. 오도메트리 전용 calibration world 추가

- 파일: `agribot_ws/src/agribot_description/worlds/odometry_calibration_world.sdf`
- 구성:
  - 평면 ground
  - sun light
  - 중심선 확인용 마커 3개
  - `agribot` 단일 배치
- 이유:
  - 기본 `farm_world.sdf`는 `greenhouse`, `tomato_plant`, `tomato`의 triangle-mesh collision 때문에
    ODE `Trimesh-trimesh contact hash table bucket overflow` 경고가 반복되어 S14P-103 검증 환경으로 부적절

### 4. 종료 안정성 정리

- 파일: `agribot_ws/src/agribot_description/agribot_description/odom_tf_broadcaster.py`
- 내용:
  - `KeyboardInterrupt`를 정상 종료로 처리
  - `rclpy.shutdown()` 중복 호출 방지

## 검증 결과

### 1. `/odom` 주기

- 명령: `timeout 10s ros2 topic hz /odom`
- 결과:
  - 평균 약 `49.09 ~ 49.13 Hz`
  - 설정한 `50 Hz`와 거의 일치

### 2. 직진 응답

- 초기 `odom -> base_link`
  - `x ≈ 0.0`
  - `y ≈ 0.0`
- `3초` 동안 `linear.x = 0.2`, `angular.z = 0.0` 전송 후:
  - `x = 0.5510845081793189`
  - `y = -1.145172846324149e-12`
- 이후 watchdog timeout이 지난 뒤 재확인:
  - `x = 0.5901500771292297`
  - 다시 재확인:
    - `x = 0.5734698196138319`
- 해석:
  - 전진 명령에 따라 `/odom`이 정상 증가
  - `y` 편차와 yaw drift는 측정 오차 수준
  - 명령 중단 뒤 위치가 계속 크게 누적되지 않아 watchdog 정지가 동작함
