# S14P-32 로봇 부품 좌표 정합

## 작업 일시

- 2026-03-17

## 작업 목표

- `map`, `odom`, `base_link`, `camera_link`, `lidar_link`, `imu_link`의 좌표 관계를 ROS 2 `tf` 트리로 일관되게 노출
- Gazebo에서 이미 발행되는 `/odom`을 RViz2와 `tf2`가 바로 사용할 수 있도록 `/tf`로 재발행
- 센서 링크의 정적 위치를 런치에서 명시해 후속 Nav2 / perception 작업의 기준선을 고정

## 프레임 구조

```text
map
└── odom
    └── base_link
        ├── camera_link
        ├── lidar_link
        └── imu_link
```

## 좌표 기준 정리

### 1. 로봇 기준축

- 기준축은 `model.sdf` 주석과 동일하게 `X=forward`, `Y=left`, `Z=up`
- 본체 기준 프레임은 `base_link`

### 2. 센서 정적 변환

| Parent | Child | Translation `(x, y, z)` | Rotation `(roll, pitch, yaw)` | 출처 |
| --- | --- | --- | --- | --- |
| `base_link` | `camera_link` | `(0.2, 0.0, 0.1)` | `(0, 0, 0)` | `model.sdf`의 `camera_link` pose |
| `base_link` | `lidar_link` | `(0.0, 0.0, 0.15)` | `(0, 0, 0)` | `model.sdf`의 `lidar_link` pose |
| `base_link` | `imu_link` | `(0.0, 0.0, 0.05)` | `(0, 0, 0)` | `model.sdf`의 `imu_link` pose |

### 3. 동적 변환

- `odom -> base_link`
  - Gazebo DiffDrive 플러그인의 `/odom` 메시지를 받아 `odom_tf_broadcaster` 노드가 `/tf`로 재발행
  - frame 이름은 `/odom` 메시지의 `header.frame_id`, `child_frame_id`를 그대로 사용

### 4. 지도 기준점

- 현 단계에서는 SLAM / localization이 아직 없으므로 `map -> odom`을 항등 정적 변환으로 발행
- 이후 `agribot_navigation`에서 실제 `map -> odom`을 발행하면 `publish_map_to_odom_tf:=false`로 비활성화해야 함

## 반영 내용

### Launch 변경

- 파일: `agribot_ws/src/agribot_description/launch/spawn_agribot.launch.py`
- 추가한 항목:
  - `publish_map_to_odom_tf` 런치 인자
  - `odom_tf_broadcaster` 노드
  - `map -> odom` 정적 TF
  - `base_link -> camera_link`, `base_link -> lidar_link`, `base_link -> imu_link` 정적 TF

  - `/odom` 구독
  - `nav_msgs/msg/Odometry`를 `tf2_ros.TransformBroadcaster`로 변환

### 패키지 의존성 보강

- 파일: `agribot_ws/src/agribot_description/package.xml`
- 추가:
  - `rclpy`
  - `tf2_ros`
  - `nav_msgs`
  - `geometry_msgs`
  - `launch_ros`
