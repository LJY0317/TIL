3. 테스트 주도적 개발 (Test Driven Development)

4. 면접 팁:
    1. "장점과 단점을 말해보세요."라는 질문을 받았을 때: 장점을 말하고, 또 장점을 말한다. "이게 단점으로도 보일 수 있겠지만 이렇게 보면 장점이 됩니다~."라는 식으로 말하기.
    2. "(해당 회사)에 관심이 많습니다"라는 말 대신 확실한 근거가 될 수 있는 말: 설립일, 대표 성함, 작년 매출, 영업 이익 등을 대답할 수 있기.
    3. CS 질문을 받았는데 대답을 못 했을 때: "죄송합니다."라고만 말하고 끝내지 말자.
        - 실무 위주 개발 경험 ("실무 위주로 개발하느라 놓쳤습니다."), "연구직이 아닌 실무자로써의 전공 지식이 우선이라 놓쳤습니다."라고 대답하기.
    4. 프로젝트 때 왜 이 포지션을 맡았는지 질문을 받았을 때: "이 포지션을 배정 받아서요."라고 대답하지 말자.
        - "전문가로써의 책임감 (납기 관리)" (할 수 있는 사람이 저밖에 없어서 제가 했습니다.), "왜 프로젝트 때 리액트를 안 썼냐" 질문 받았을 때, "프로젝트에만 안 썼을 뿐이지 당연히 프론트엔드를 선호하는 개발자이기 때문에 리액트를 공부했고 알고는 있습니다."라고 대답할 수 있기. "아무도 안 해서요."라고 대답하지는 말자.
        - "현업이라 생각하고 리스크 관리를 위해 이 포지션을 했습니다."
    5. 면접은 기술이 아니다:
        - 단순히 대답을 계속 단답식으로 하면, 면접관도 이 사람한테 더 질문하기 싫어진다. "아 질문 주셔서 감사합니다. (대답 끝) 그리고 어쩌구 저쩌구..." 등등 더 간절한 모습이 느껴지면 좋을 것이다.

2. 실제 프로젝트에서도 갈등이 발생하는 건 당연.

3. 프로젝트 기본 관리 방안:
    1. 업무 분해: 프로젝트 진행 과정에서 관리 관점에서 업무를 상세하게 분해하는 과정.
        > 분석/설계 -> 디자인 -> 개발 -> 테스트 -> 오픈 -> 안정화
    
    2. 규칙 관점 관리: 조직이 가지는 문화를 기반으로 문제 대응
        > 의견 충돌 발생시 [OO]의 중재를 통한 해결.

    3. 기술 리더를 한 명 뽑아두면 편하다..

    4. 관리의 기본은 [팀장 + 개발 리더]가 진척이나 문제 상황을 파악해야 하는 것이 아니라 개별 구성원이 [진척 상황, 문제 상황, 변경 사항] 등을 공유하는 것임을 인지해야 함.

    5. [권한]은 [규칙]으로 정의되지 않는 예외적인 문제 발생시 해결.


### 2026. 01. 15.
비기능 요구사항 정의 (예시)
    > 시스템의 성능, 보안, 사용성 등 품질 특성이나 제약 조건
1. 강의 신청 기간 중 원활한 진행.
2. 언제 어디서든 망 구분 없이 접근 가능한 환경.
3. PC 및 모바일 플랫폼 지원.
4. 유저 별 Permission 부여.
5. 이슈 사항에 대한 데이터 손실 최소화.

강의 신청 기간 중 원활한 진행 (예시)
- Source: 학생 N명.
- Stimulus: 로그인을 시도하며 비정상적인 접근 발생.
- Environment: 학기 시작 전날, 피크 시간.
- Artifact: 수강 신청 시스템.
- Response: 먼저 도착한 순서대로 요청 처리
- Response measure: 각 요청 2초 이내 처리 완료.

유저 별 Permission 부여 (예시)
- Source: 학생 또는 외부 사용자.
- Stimulus: 로그인을 시도하며 비정상적인 접근 발생.
- Environment: 외부 네트워크를 포함한 접속 환경.
- Artifact: 학사 관리 시스템.
- Response: 비정상 접근을 차단하고 알림 로그를 생성.
- Response measure: 차단 시간은 1초 이내.

가용성 부분 (예시)
- Source: 내부 시스템 모듈.
- Stimulus: 등록된 서비스 인스턴스가 사라짐.
- Environment: 고기용성 구성이 된 클러스터 환경.
- Artifact: 서비스 디스커버리 매커니즘
- Response: 해당 인스턴스를 트래픽 대상에서 제외.
- Response measure: failover는 5초 이내에 완료.

품질 속성 시나리오를 쓰고, 기술 스택을 쓴다.
