### 2026. 03. 25.
## Chrome / Electron 계열 앱에서 한글 입력이 씹히거나 두 번 들어갈 때 해결한 내용

### 증상
- Google Chrome의 Google Docs, Google Keep에서 한글 입력 중 글자가 가끔 안 들어가거나, 방금 쓴 글자가 지워지는 현상이 있었다.
- 터미널에서 한글을 입력할 때 같은 글자가 2번 들어가는 경우가 있었다.
- Antigravity 오른쪽 사이드바에서 한글 입력을 시작하면 2번째 글자를 칠 때 1번째 글자까지 같이 사라지는 현상이 있었다.

### 원인 요약
- 이 문제는 GPU 문제가 아니라 `한글 입력기(IME)`와 `Chromium / Electron 계열 입력창`의 조합 문제에 가깝다.
- 실제 환경을 확인해 보니 현재 시스템은 `Ubuntu 24.04.4 + GNOME(X11) + Chrome 146 + Antigravity(Electron)` 조합이었다.
- 가장 큰 문제는 입력기 설정이 서로 충돌하고 있었다는 점이다.
  - `~/.xinputrc`는 이미 `fcitx5`를 사용하도록 설정되어 있었다.
  - 그런데 `~/.profile`, `~/.bashrc`에서는 `GTK_IM_MODULE=ibus`, `QT_IM_MODULE=ibus`, `XMODIFIERS=@im=ibus`를 강제로 넣어 두고 있었다.
- 즉, 설정 파일 하나는 `fcitx5`를 가리키고, 실제 세션 환경은 `ibus`를 가리키는 모순 상태였다.
- 이런 상태에서는 Chrome, Electron, 터미널처럼 입력 이벤트 처리가 예민한 앱에서 한글 조합 문자열(preedit)이 꼬일 수 있다.

### 실제로 적용한 수정
#### 1. 셸 / 데스크톱 세션 입력기 환경을 `fcitx5`로 통일
- `~/.profile` 수정
```bash
export GTK_IM_MODULE=fcitx
export QT_IM_MODULE=fcitx
export XMODIFIERS=@im=fcitx
```

- `~/.bashrc` 수정
```bash
export GTK_IM_MODULE=fcitx
export QT_IM_MODULE=fcitx
export XMODIFIERS=@im=fcitx
```

- 데스크톱 세션용 환경 파일 추가
  - `~/.config/environment.d/90-input-method.conf`
```bash
GTK_IM_MODULE=fcitx
QT_IM_MODULE=fcitx
XMODIFIERS=@im=fcitx
```

#### 2. 로그인 시 `fcitx5`가 자동 실행되도록 설정
- `~/.config/autostart/org.fcitx.Fcitx5.desktop` 추가
- 핵심은 로그인할 때 `fcitx5`가 자동으로 떠서, 새로 실행하는 Chrome / Antigravity / 터미널이 같은 입력기 환경을 공유하게 만드는 것이다.

#### 3. GNOME이 GTK 입력 모듈로 `fcitx`를 쓰도록 명시
```bash
gsettings set org.gnome.settings-daemon.plugins.xsettings overrides "{'Gtk/IMModule': <'fcitx'>}"
```

#### 4. 현재 `ibus-hangul` 쪽 즉시 완화 설정도 함께 적용
- `IBus Hangul`의 `use-event-forwarding` 옵션을 껐다.
- 이 옵션은 어떤 앱에서는 workaround처럼 동작하지만, Chromium / Electron 계열에서는 오히려 입력 중복이나 조합 꼬임처럼 보일 수 있다.
```bash
gsettings set org.freedesktop.ibus.engine.hangul use-event-forwarding false
```

#### 5. 현재 로그인 세션에도 바로 반영
```bash
systemctl --user import-environment GTK_IM_MODULE QT_IM_MODULE XMODIFIERS
dbus-update-activation-environment --systemd GTK_IM_MODULE QT_IM_MODULE XMODIFIERS
fcitx5 -d -r
```

### 수정 후 확인한 내용
- 세션 환경변수는 아래처럼 정리되었다.
```bash
GTK_IM_MODULE=fcitx
QT_IM_MODULE=fcitx
XMODIFIERS=@im=fcitx
```

- 실행 중인 입력기는 `fcitx5`만 남도록 정리되었다.
- `fcitx5-remote` 기준으로 한글 입력기(`hangul`) 전환도 정상 동작하는 것까지 확인했다.

### 사용자가 꼭 해야 하는 마무리
- 이미 열려 있던 Chrome, Antigravity, 터미널은 예전 `ibus` 환경을 물고 있을 수 있으므로 완전히 종료 후 다시 실행해야 한다.
- 가장 확실한 방법은 `로그아웃 -> 다시 로그인`이다.
- 한/영 전환은 기존처럼 `오른쪽 Alt(Hangul 키)`를 사용하면 된다.

### 핵심 결론
- 이번 문제의 본질은 "크롬이 이상한 것" 하나로 끝나는 문제가 아니라, `fcitx5`와 `ibus`가 섞여 있는 입력기 환경 충돌이었다.
- 해결 방향은 "입력기 스택을 하나로 통일"하는 것이고, 이번에는 `fcitx5` 기준으로 정리했다.
- 같은 증상이 다시 생기면 가장 먼저 `echo $GTK_IM_MODULE $QT_IM_MODULE $XMODIFIERS`로 현재 세션이 정말 `fcitx fcitx @im=fcitx`인지 확인하면 된다.
